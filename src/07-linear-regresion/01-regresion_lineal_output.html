Introducción¶

En este trabajo, usamos datos asociados a todos los alumnos de Grado en Ingeniera Telemtica y Grado en Ingeniera de Sistemas de Telecomunicacin de la UPCT, que hayan superado 120 ECTS, que provengan de la Regin de Murcia y que se hayan examinado de Fsica y Matemticas_II en la Prueba de Acceso a la Universidad (PAU).

Objetivo:¶

Nuestro objetivo es estudiar la posibilidad de predecir la nota media a partir de algunos datos en el ingreso del estudiante (calificacin y ranking PAU, as como de Fsica y Matemticas II) y de sus resultados en algunas de las asignaturas ms exigentes de la titulacin:

  1. Fundamentos de programacin.
  2. Sistemas y circuitos
  3. Sistemas lineales
  4. Ondas electromagnticas

Implementación de la función coste.¶

Recordar que la funcin coste es (ver transparencias): $$J(\theta)=\frac{1}{n}\sum_{i=1}^n\left(y_i-x_{i\bullet}^T\theta\right)^2=\frac{1}{n}\lvert\lvert \mathbf{y}-\mathbf{X}\theta\rvert\rvert^2=\frac{1}{n}\left(\mathbf{y}-\mathbf{X}\theta\right)^T\cdot \left(\mathbf{y}-\mathbf{X}\theta\right),$$ donde $\mathbf{y}$ es el vector que contiene todas las observaciones de la variable respuesta, y la matriz $\mathbf{X}$ la matriz de diseo. La matriz de diseo contiene los valores de las caractersticas que queremos tener en cuenta en nuestro modelo para todos los individuos y una columna de 1.

Definir el array de numpy y que contenga los valores de MEDIA, y el array X que sea la matriz de diseo asociada a la frmula:

$$MEDIA=\theta_0+\theta_1 NOTA\_PAU\_CALIFICACION$$

pero habiendo, en primer lugar, quitado las filas que tienen un dato faltante en alguna de estas dos columnas

Para definir X, habr que usar np.concatenate, y np.ones tal como est explicado en las transparencias del tema.

Debemos monitorizar el algoritmo del gradiente¶

Para comprobar cmo evoluciona el algoritmo del gradiente y en particular, si hemos escogido bien el valor de $\alpha$, es importante comprobar la evolucin del valor de la funcin coste con las iteraciones.

Para ello vamos a introducir un DataFrame llamado monitor, que recoja los valores de alpha, theta, J(theta, X, y) y gradJ(theta, X, y) a medida que vamos iterando el algoritmo.

Empezamos por definir dos funciones que faciliten el manipular el dataframe monitor:

  • la primera funcin inicializa_monitor que devuelve un dataframe preparado con las columnas necesarias y con el tipo apropiado.
  • la segunda funcin inserta_en_monitor, que aade a monitor una fila con los valores de iteracion, alpha, theta y el coste.

# Nada que completar pero s entender el cdigo
def inicializa_monitor():
    monitor = pd.DataFrame(
        {
            'alpha': float(), 
            'theta_0': float(), 
            'theta_1': float(), 
            'coste': float()
        }, 
        index=[]
    )
    return monitor
# ------------------------------------------------------------------------------------------------------
def inserta_en_monitor(monitor, iteracion, alpha, theta, coste):
    fila = pd.DataFrame(
         {
             'alpha': alpha, 
             'theta_0': theta[0, 0], 
             'theta_1': theta[1, 0], 
             'coste': coste
        }, 
        index=[iteracion]
    )
    return pd.concat([monitor,fila])

Tenis ahora que aprovechar vuestra implementacin del algoritmo del gradiente para crear una nueva funcin buscar_optimo_gradiente que admita los siguientes parmetros:

  • theta_inicial: un vector numpy de valores para los componentes de theta_inicial
  • coste: la funcin de coste definida, que tome como parmetro theta
  • gradiente: la funcin que calcule el gradiente de la funcin de coste
  • alpha: el paso escogido para cada iteracin
  • iteraciones: el nmero total de iteraciones que se realiza hasta parar y devolver el ptimo
  • monitor: el dataframe que contiene la evolucin de las cantidades de inters durante la iteracin.
